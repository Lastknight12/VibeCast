# Запуск проєкту

## 1. Змінні середовища (`.env`)

Перед запуском необхідно підготувати файли змінних середовища.

### Клієнт

Для клієнтської частини можна використати вже готовий файл: `./client/.env`.

### Сервер

Для серверної частини можна використати вже підготовлений `.env` файл з Google та Cloudinary ключами, які вже там прописані.

---

## 2. Локальна IP-адреса

Для коректної роботи **SFU-сервера** в локальній мережі необхідно вказати **локальну IP-адресу** машини у файлі: `./server/.env`

### Windows

1. Натиснути **Win + R**
2. Ввести `cmd` та натиснути **Enter**
3. У терміналі виконати `ipconfig`
4. Знайти рядок **IPv4 Address** (наприклад `192.x.x.x` або `10.x.x.x`)
5. Записати цю адресу в `./server/.env` в змінну `ANNOUNCED_IP`

### Linux

1. Відкрити термінал
2. Виконати команду `ip a`
3. Знайти інтерфейс зі статусом **UP**, який відповідає поточній мережі
4. Скопіювати IPv4-адресу (`inet x.x.x.x`)
5. Записати її в `./server/.env` в змінну `ANNOUNCED_IP`

---

## 3. Розгортання проєкту

### 3.1 Запуск через Docker (рекомендовано)

У головній папці проєкту виконати:

```bash
docker compose up -d
```

Ця команда збілдить клієнт і сервер та запустить всі сервіси. Після цього клієнт буде доступний за адресою: `http://localhost:3000`

### 3.2 Ручний запуск

#### 3.2.1 Клієнт

У папці `client` виконати:

```bash
npm ci
npm run build
node .output/server/index.mjs
```

#### 3.2.2 Сервер

У папці `server` виконати:

```bash
npm ci
npm run build
npm run start
```

Таким чином клієнт стане доступний за адресою `http://localhost:3000`
